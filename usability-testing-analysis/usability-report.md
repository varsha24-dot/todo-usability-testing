# Usability Testing and Analysis of a To-Do List Web Application

---

# 1. Introduction

This project focuses on conducting usability testing and analysis of a To-Do List Web Application. The application helps users manage daily tasks efficiently through features such as adding tasks, deleting tasks, marking tasks as completed, and filtering tasks.

The main purpose of this project is to evaluate the usability, accessibility, and user experience of the application through real user testing and iterative improvements.

---

# 2. Objectives

The objectives of this project are:

- To develop a simple and user-friendly To-Do List application
- To conduct usability testing with real users
- To identify usability issues and user difficulties
- To collect user feedback and observations
- To improve the application design based on testing results

---

# 3. Technologies Used

- HTML
- CSS
- JavaScript
- Local Storage
- Visual Studio Code

---

# 4. Features of the Application

The application includes the following features:

- Add tasks
- Delete tasks
- Mark tasks as completed
- Filter tasks (All / Completed / Pending)
- Responsive and clean user interface
- Local storage support

---

# 5. Usability Test Plan

## Participants

A total of 5 users participated in the usability testing process.

## Tasks Given to Users

1. Add a new task
2. Mark a task as completed
3. Delete a task
4. Filter completed tasks
5. Filter pending tasks

## Metrics Collected

- Time taken to complete tasks
- Ease of navigation
- Number of errors
- User satisfaction
- Suggestions and feedback

---

# 6. Testing Methodology

Users were asked to interact with the application independently. During testing, user interactions, mistakes, confusion points, and completion times were observed and recorded.

Feedback was collected after task completion through informal questioning and observation.

---

# 7. Observations

| User | Observation |
|------|-------------|
| User 1 | Initially confused about clicking task text to mark completed |
| User 2 | Delete button appeared small |
| User 3 | Requested darker text for readability |
| User 4 | Navigation was simple and easy |
| User 5 | Suggested adding task counter |

---

# 8. User Feedback

## Positive Feedback

- Interface is clean and simple
- Application is easy to use
- Tasks can be managed quickly
- Filter options are useful

## Suggestions for Improvement

- Add task counter
- Improve button visibility
- Add dark mode
- Add task editing feature

---

# 9. Improvements Made After Testing

The following improvements were implemented after usability testing:

- Increased delete button size
- Added instruction text for completing tasks
- Improved spacing and button design
- Enhanced overall UI appearance
- Improved input field visibility

---

# 10. Results and Analysis

The usability testing process showed that users were able to complete most tasks successfully with minimal errors. The interface was considered beginner-friendly and easy to navigate.

Minor usability issues were identified and corrected through iterative improvements. Overall user satisfaction was positive.

---

# 11. Conclusion

The usability testing process helped identify important user experience issues and provided valuable insights into improving interface design.

This project successfully demonstrated the importance of usability evaluation, feedback collection, and iterative design improvements in web application development.

The final application became more user-friendly, accessible, and efficient after incorporating user feedback.
